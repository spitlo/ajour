More recently I've run into the problem that when I load new production data into tables with auto increment primary key columns, I'll get a conflict, i.e. I load some production data in locally, and then when the auto increment runs it'll hit a conflict - because my local sequences are completely out of sync with production. The solution is to update my local sequence value. This is a two part process. 1. Update sequences locally: ALTER SEQUENCE __seq RESTART WITH ; It's important that I update to the next value, not the current value: ALTER SEQUENCE user_id_seq RESTART WITH 305; Now when the next id for the user is loaded through an insert, the sequences are in sync.
